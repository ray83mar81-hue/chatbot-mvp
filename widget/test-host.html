🧪 Test adversario del widget

Esta página inyecta CSS agresivo y recursos pesados para verificar que el widget sigue funcionando aislado. Abre la consola (F12) para ver warnings.

¿Qué debería funcionar?

  • El botón flotante del chat debería verse correctamente, sin heredar el rojo / amarillo / borde morado del host.
  • Su ventana debería abrirse con sus colores de marca, no con los del CSS agresivo.
  • El input del chat no debe ser rosa con borde naranja.
  • Los SVG internos no deben llevar el fondo cian de arriba.
  • No debe haber flash of unstyled content (FOUC) perceptible al cargar.
  • El widget debe aparecer aunque el script se declare con async o defer, o aunque esté en <head>.

¿Qué comprobar en consola?

  • ❌ Warning de Synchronous XHR → no debería salir
  • ❌ document.currentScript is null → si sale, el script no se auto-localiza bien con async/defer
  • ❌ Cannot read properties of null (reading 'appendChild') → problema con body no listo
  • ✅ Una sola petición a /business/1/languages. Si hay dos, hay refetch innecesario.

Cómo probar otras configuraciones

  1. Cargar el script con async
  2. Cargar el script con defer
  3. Cargar el script en <head>
  4. Forzar idioma inglés
  5. Móvil: abre con DevTools → modo responsive → iPhone 12 → pulsa el input del chat → verifica que el teclado no tapa el input
